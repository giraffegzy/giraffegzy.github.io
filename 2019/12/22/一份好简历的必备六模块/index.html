一、写出好简历的必备六模块

简历作为一个走向社会的必备任务道具，其质量好坏直接决定了你是不是有机会获得你想要的offer。

所以我们需要好好打磨自己的简历。一份好的简历在框架上一定包含以下六个模块，由于自己是产品经理，所以举的例子都和产品有关嗷。

  • 基础信息
  • 教育经历
  • 工作经历
  • 项目经历
  • 个人技能
  • 个人评价

接下来我们再来仔细聊一聊这六个模块分别包含那些信息和需要注意什么？

1.基础信息

如果你投出去了一份没有基础信息的简历，基本上等于没投。因为无论后面写得多么好，也不知道这份简历是谁的。所以基础信息的重要性不言而喻了。

那么，以下内容是你必须要包含的：

  • 姓名
  • 联系方式（电话、微信、邮箱）
  • 出身年月
  • 求职意向
  • 照片（根据求职岗位来决定是否需要）
  • 关键词（如果能够准确提供的话）
  • 身高（根据求职岗位来决定是否需要）
  • 到岗时间（根据求职时间来决定是否需要）

在基础信息这里也有一些值得注意的点：

  1. 确认联系方式是否正确
  2. 求职意向不要同时填写多个岗位
  3. 如果要有照片，请一定使用证件照，而非生活照、大头照

2.教育经历

在完成基础信息的构建之后，紧接着我们应该准确的描述自己的教育经历。你可以看到大部分公司的JD上面，第一条通常都是针对学历的限制。所以只有你准确的描述了自己的教育经历之后，才能节约双方的时间，避免因为错误信息，浪费双方的时间。

你需要注意这些点：

  • 按时间倒序进行填写（如无特殊需求，一般从大学开始填写）
  • 要准确的填写学校全称而非英文简称
  • 如果不是校招，学历不是加分项的话，那么我们可以考虑把学历放在六要素的最后，或者倒二去填写。

格式如下：

XXX大学 2016.9——2020.6
软件工程专业 本科

3.工作经历

对于工作经历而言，你需要告诉招聘方和面试官，你的工作大概履历是什么，你做过什么，以及让招聘方知道你经验和年龄是不是大概符合招聘的要求。

不要写太详细的工作内容，用一句话讲明白这段时间内，作为什么角色，完成了什么工作。

你需要注意这些点：

  • 按时间倒序进行填写（如果跳槽特别频繁，那可能是个坏消息）
  • 要准确的填写公司全称而非简称
  • 请用数字来量化你的工作成果，或是有实例能够佐证你的确做了这些事情。
  • 如果没有非常确认的实例可以佐证的成果，尽量不要填写。

格式如下：

XXX公司 2016.9——2020.6
产品经理
1、负责xxx产品／项目，完成xxxx产品／项目从0到1的规划与落地
2、负责xxx优化项目
3、负责编制xxxx报告，协助搭建xxx模型

4.项目经历

搞定了工作经历之后，招聘方会大致了解你做过些什么，现在需要详细的告诉他们，我们有什么样的能力，我们能够为公司带来一些什么。
我们可以按照时间(倒序)-项目-角色的格式来书写，内容上我们将项目经历可以拆分成三个子模块分别是：项目简介、工作内容、工作业绩

  • 项目简介
    项目简介需要简短、概括的讲明白你做的项目是什么，这样招聘方才知道你的项目经历是不是能够与当前招聘的岗位匹配。
  • 工作内容
    工作内容主要是将工作抽象成一个一个的关键词，通过关键词组合来进行精炼，再展开来讲具体工作
  • 工作业绩
    工作业绩就是所谓的成果，可以从基础建设、效率提升、数据指标 三方面入手。
  • 基础建设*，主要是从产品目标或者产品战略价值入手。比如实现了XXX业务闭环，为XXX提供了支持（这样能表达清楚自己是否对于产品上下游有一个较为全面的认识）
  • 数据指标*，主要是从产品本身的指标出发，去突出重点指标因为你的工作而产生了变化，比如日活、拉新率、留存、主流程转化、传播率等等

格式如下：

XXX公司 2016.9——2020.6
产品经理
【项目简介】XXXapp是一款面向xxxx的xxxx产品，主要功能有xxx
【工作内容】
1、从0到1：作为产品负责人，从0到1规划与搭建xx产品，完成了xxxx、xxx、xxx等功能
2、流程与逻辑设计：通过xxxxx（如分析用户行为／消费习惯；深入接触业务）制定xxxx流程，搭建xxxx体系
3、跨部门沟通：与多个部门沟通协作，解决了xxxx，完成了xxxx
4、数据分析：怎么分析？什么数据？解决什么问题？
5、前端与后台业务：balabal
【工作业绩】
1、（产品目标是什么）达成情况如何？
2、（产皮战略价值是什么）给各干系人带来了什么价值
3、数据指标提升情况如何

5.个人技能

这个模块就不多说了，把你行业相关、必备且具有权威性的奖项列上去就好了。

6.个人评价

这个部分就比较主观了，需要提炼出自己的一些优势比如：

  • 拥有X年XXX岗位经验
  • XXXX业务和行业十分了解，XXX产品重度使用者，使用XXX产品XXX小时
  • 拥有较强的逻辑性，负责过XXX负责流程设计
  • 拥有数据思维，（实例证明）

总而言之就是帮助招聘方来快速画像你这个面试者，同样可以提供面试时候的一些谈资，来帮助面试官更好的了解你。

小Tips

1.简历一定要精炼，能够重点突出你想要表达的东西，同时要让人通过简历理解你的工作，哪怕是行业不相关的人。
2。对于简历中的任何部分都有非常熟悉，能够有实例来佐证。
3、不要伪造经历，做一个诚实的人，才会拥有好运

最后，这是我第一年踏入社会，希望未来有一天能够进入梦想的公司，做喜欢的产品，并为之感到幸福。

产品经理就是以产品为笔，与世界对话。——俞军
